# Georgia Flow – რეკლამის ბლოკები დიზაინში + მონეტიზაციის სისტემა

## მიზანი
- რეკლამა უნდა იყოს დისკრეტული, არა შემაწუხებელი (native-style)
- ლოკალური ბიზნესებისთვის (რესტორნები, ტურები, კაფეები, ღვინის მარანები)
- ადმინის პანელიდან სრული მართვა
- მომავალში: ფასიანი სპონსორობა + კომისია ბუკინგებზე

## რეკლამის ტიპები და დიზაინი
1. **Sponsored Card** (ყველაზე მნიშვნელოვანი, native)
   - გარეგნობა: იგივე როგორც ჩვეულებრივი Place/Event კარდი, მაგრამ:
     - მცირე ლეიბლი ზედა მარჯვნივ: "Sponsored" ან "რეკლამა" (ნაცრისფერი, პატარა font)
     - Border: ოდნავ გამოკვეთილი ნარინჯისფერი (#FF9800 1px)
   - კონტენტი: ბიზნესის ფოტო, სახელი, მოკლე აღწერა ("20% ფასდაკლება დღეს"), CTA button ("ნახე" ან "დაჯავშნე")
   - პოზიცია: Home Feed-ში ყოველ 3-5 ორგანულ კარდს შორის (random ან admin-ის მიერ დაყენებული)

2. **Banner** (ჰორიზონტალური)
   - ზომა: 320x100px ან full-width მობილურზე
   - დიზაინი: მუქი ლურჯი gradient (#0D47A1 → #1976D2), თეთრი/ნარინჯისფერი ტექსტი
   - ელემენტები: ლოგო + ტექსტი + CTA button ("შეიტყვეთ მეტი")
   - პოზიცია:
     - Home page-ის ქვედა ნაწილში (scroll-ის ბოლოს)
     - Place/Event დეტალურ გვერდზე ქვედა სექციაში (რუკის ქვემოთ)
     - Profile-ის ქვედა ნაწილში (optional)

3. **Interstitial / Pop-up** (მხოლოდ მომავალში, არა MVP-ში)
   - გახსნისას ან ყოველ 5-10 წუთში ერთხელ (მაგრამ მხოლოდ უფასო უსერებისთვის)
   - დახურვის ღილაკი აუცილებელი
   - დიზაინი: მსგავსი banner-ის, მაგრამ დიდი

## Ads Table Supabase-ში (ადმინის მენეჯმენტისთვის)
```sql
create table ads (
  id bigint generated by default as identity primary key,
  title text not null,
  description text,
  image_url text,          -- Supabase Storage
  link_url text,           -- external link ან internal place_id
  position text check (position in ('home_feed', 'place_detail', 'event_detail', 'profile')), -- სად გამოჩნდეს
  type text check (type in ('sponsored_card', 'banner')),
  start_date date,
  end_date date,
  active boolean default true,
  clicks integer default 0,
  impressions integer default 0,
  created_by uuid references profiles(id), -- admin ან business user მომავალში
  created_at timestamptz default now()
);

-- RLS: მხოლოდ admin ხედავს/ცვლის
alter table ads enable row level security;
create policy "Admin full access to ads" on ads
  using (exists (select 1 from profiles where id = auth.uid() and role = 'admin'));
ადმინ პანელში Ads მენეჯმენტი

Ads ტაბი sidebar-ში:
ლისტი ტაბლა: title, type, position, active status, dates, impressions/clicks
"ახალი რეკლამის დამატება" button → modal/form:
Title, Description
Image upload (Supabase Storage)
Link (external ან place/event select)
Position select (dropdown)
Start/End date (date picker)
Active toggle

Edit/Delete buttons თითო რეკლამაზე
Stats: total impressions, clicks, CTR (clicks/impressions)


Frontend Integration (როგორ გამოჩნდეს)

Home Feed-ში: როცა fetch-ავ places/events-ს, merge ads-ს random პოზიციებზე (მაგ. every 4th item insert sponsored card)
Place/Event გვერდზე: ქვედა სექციაში <AdBanner /> component თუ არსებობს active ad იმ position-ზე
Component მაგალითი:tsxfunction SponsoredCard({ ad }) {
  return (
    <div className="border border-orange-500 rounded-lg p-4 bg-white">
      <span className="text-xs text-gray-500">რეკლამა</span>
      <img src={ad.image_url} alt={ad.title} className="w-full h-48 object-cover" />
      <h3>{ad.title}</h3>
      <p>{ad.description}</p>
      <button className="bg-orange-500 text-white px-4 py-2 rounded">ნახე</button>
    </div>
  );
}
